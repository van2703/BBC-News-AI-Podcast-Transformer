BBC News AI Podcast Transformer

Detailed System Design  

Version 1.0  
Developer: Vân Phạm

# 1\. Introduction

This document describes the Detailed Design of the BBC News AI Podcast Transformer system.  
The system automatically collects news articles from BBC News, processes them using artificial intelligence,  
generates a podcast script, converts the script into audio, and publishes the podcast on a web interface.

The purpose of this document is to describe the internal design of each component, the data flow between modules,  
and the technical implementation details required to build the system.

# 2\. System Overview

The system operates as an automated pipeline that transforms news articles into a podcast episode.

Main pipeline:

1\. Fetch news articles from BBC.  
2\. Process articles using an AI engine.  
3\. Generate a podcast script.  
4\. Convert the script into speech.  
5\. Add background music based on sentiment.  
6\. Store the audio file.  
7\. Save metadata in the database.  
8\. Display the podcast on a website.

The system runs automatically every day using a scheduler.

# 3\. System Architecture Modules

The system is divided into several modules:

1\. News Fetcher Module  
2\. AI Script Generator Module  
3\. Podcast Generator Module  
4\. Storage Manager Module  
5\. Database Manager Module  
6\. Scheduler Module  
7\. Web Interface Module

Each module is designed to be independent and responsible for a specific task in the pipeline.

# 4\. News Fetcher Module

Purpose:  
Collect news articles from BBC News.

Inputs:  
BBC RSS feeds or article URLs.

Outputs:  
A list of article objects containing title, URL, and content.

Key Functions:  
\- fetch\_bbc\_articles()  
\- parse\_article()  
\- clean\_article\_text()

Process:  
The module reads RSS feeds from selected BBC sections, extracts article links,  
downloads the article pages, and extracts the textual content.  
The cleaned text is returned to the AI Script Generator module.

# 5\. AI Script Generator Module

Purpose:  
Transform news articles into a podcast-ready script.

Inputs:  
Raw article text from the News Fetcher.

Outputs:  
Podcast script and sentiment label.

Key Functions:    
\- summarize\_article()  
\- generate\_podcast\_script()  
\- analyze\_sentiment()

Process:  
The system sends the article text to the AI model (openrouter API).  
The AI summarizes the article, and formats the text into a conversational podcast script.  
Sentiment analysis is also performed to determine the tone of the episode.

# 6\. Podcast Generator Module

Purpose:  
Convert the podcast script into audio.

Inputs:  
Podcast script text and sentiment label.

Outputs:  
Podcast audio file (.mp3)

Key Functions:  
\- text\_to\_speech()  
\- select\_background\_music()  
\- mix\_audio\_tracks()  
\- export\_audio\_file()

Process:  
The text-to-speech engine converts the script into spoken audio.  
Background music is selected depending on the sentiment.  
MoviePy is used to combine voice and music into a final podcast file.

# 7\. Storage Manager Module

Purpose:  
Store and manage generated podcast files.

Inputs:  
Audio file generated by the Podcast Generator.

Outputs:  
Public URL to the podcast audio file.

Key Functions:  
\- upload\_to\_drive()  
\- delete\_old\_files()  
\- generate\_public\_link()

Process:  
The system uploads the generated podcast to Google Drive using the Drive API.  
Files older than the retention policy are automatically deleted to save storage.

# 8\. Database Design

Database: SQLite

Episodes Table:  
\- id  
\- title  
\- script  
\- bbc\_links  
\- audio\_url  
\- sentiment  
\- created\_at

Logs Table:  
\- id  
\- message  
\- level  
\- created\_at

Purpose:  
The Episodes table stores podcast metadata.  
The Logs table records system events and errors for monitoring.

# 9\. Scheduler Module

Purpose:  
Automate the execution of the podcast generation pipeline.

Tools:  
Python schedule library or cron jobs.

Process:  
Every day at 06:00 AM:  
1\. Run the news fetching module.  
2\. Generate podcast script.  
3\. Produce audio.  
4\. Upload audio to storage.  
5\. Save episode metadata to the database.  
6\. Update the website interface.

# 10\. Web Interface (Streamlit)

Purpose:  
Provide a user-friendly interface for listening to podcasts and managing the system.

User Features:  
\- View latest podcast episodes.  
\- Play audio using a built-in audio player.  
\- Download podcast files.  
\- Search episodes.

Admin Features:  
\- Manually trigger podcast generation.  
\- Delete episodes.  
\- View system logs.

# 12\. System Pipeline Execution

The system pipeline is controlled by the main program.

Execution Steps:

1\. Fetch news articles.  
2\. Generate podcast script using AI.  
3\. Convert script to speech.  
4\. Add background music.  
5\. Export final audio file.  
6\. Upload audio to cloud storage.  
7\. Save metadata in the database.  
8\. Display the podcast on the website.

This pipeline ensures the full automation of the podcast generation process.

# 13\. Non‑Functional Considerations

Performance:  
The system should generate a podcast episode within a reasonable time (10–20 minutes).

Reliability:  
If no new articles are available, the system selects evergreen content.

Security:  
API keys must be stored securely and never exposed in public repositories.

Cost Optimization:  
Low-cost or free APIs are prioritized for TTS and AI processing.

Maintainability:  
The modular architecture allows future improvements and feature extensions.

# 14\. Conclusion

The Detailed Design document provides a clear technical description of the BBC News AI Podcast Transformer.  
It defines the modules, data flow, database structure, and system execution pipeline.

This design ensures that the system is scalable, maintainable, and suitable for implementation using Python,  
AI APIs, and lightweight web technologies such as Streamlit.
